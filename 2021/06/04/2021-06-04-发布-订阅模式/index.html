发布-订阅模式
Eli Lv4
  2021-06-04 21:25:21 2021-06-04 21:25  
  • 设计模式 
 
  • 设计模式 

javascript 中的发布-订阅模式其实很常见，典型例子就是对 dom 节点添加事件监听。

概念

简单来说，它定义对象间的一种一对多的依赖关系，当一个对象的状态发生改变时，所有依赖于它的对象都将得到通知。例如为某个 dom 节点添加事件监听，并传递相应的处理函数，这样就完成了订阅；等到浏览器检测到该 dom 节点有对应事件触发，就会对用户发布这一信息，并执行先前传递的处理函数，这就是一个完整的发布-订阅流程。

优点

  1. 对象间解耦，降低了对象间的耦合度
  2. 时间上的解耦，可以应用于异步编程中替代回调函数（例如监听XHR对象的readyStateChange事件）

实现方式

  1. 首先需要一个发布者对象
  2. 发布者对象需要维护一个缓存列表，用于存储回调函数以通知订阅者
  3. 发布消息时，发布者遍历缓存列表，依次触发其中的订阅者回调函数

发布-订阅模式与观察者模式经常被混为一谈，实际上两者有细微的差别具体参考文章
