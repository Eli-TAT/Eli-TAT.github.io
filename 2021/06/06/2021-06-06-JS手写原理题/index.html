汇总一些 JS 手写题，偏向于理解原理后实现某些 JS 内置 API。

1.手写一个 new 运算符

new 的过程：

  1. 创建一个空对象；
  2. 空对象的proto属性（IE 没有该属性）指向构造函数的原型对象 prototype（链接原型）；
  3. 构造函数的 this 指向空对象（绑定 this）；
  4. 执行构造函数中的代码，为空对象添加属性、方法；
  5. 若构造函数返回一个基本类型，那么返回创建的空对象；若构造函数返回一个引用类型，返回值将覆盖之前创建的空对象；

2. 实现 instanceof 运算符

instanceof 运算符用于检验构造函数的 prototype 是否出现在某个对象的原型链上，从而可以判断对象的类型。根据原型链，原型之上可能还存在原型，因此实现的关键在于还要上溯原型链。

isPrototypeof()与instanceof的区别在于，它用于测试一个对象是否存在于另一个对象的原型链上。用法参考

3. 手写 call、apply、bind

  • call()实现

简单来说，我们需要在 Function 的原型上添加一个 call 方法，这个方法要能够为调用 call 的函数实例指定 this，并能传递参数，最后执行函数。解决方法就是，将调用 call 的函数传入指定的函数上下文，执行函数并返回结果。

  • apply()实现

apply()与 call()唯一的区别在于传递参数的形式

  • bind()实现

bind()返回一个函数的拷贝，并能设置 this 值和初始参数。由于 bind 要考虑两种情况（返回普通函数或构造函数），而且还要涉及到参数合并，因此实现要复杂一些。

5. Object.is()实现

Object.is()方法用于判断两个值是否相等，并且比较时不会对两者进行类型转换，与===全等运算符十分相似，但它们之间还是存在一些区别

  • NaN 在===中是不相等的，而在 Object.is 中是相等的;
  • +0 和-0 在===中是相等的，而在 Object.is 中是不相等的;

6. Object.create()实现

Object.create()方法用于创建一个对象，并可以为其指定原型和属性，其中Object.create(null)创建的对象既没有属性也没有原型。用法参考

7. 用 setTimeout 模拟实现 setInterval

为什么要用 setTimeout 模拟实现 setInterval？
根据事件循环的知识，setInterval 定时器 属于宏任务，它设置的延迟时间并不是回调执行的时间间隔，而是回调进入任务队列的事件间隔，可能存在以下情况：
1）前面的任务较多或者执行时间较长，导致定时器的回调还在执行过程中，下一个回调紧接着进入了任务队列，这样就会造成定时器执行间隔不准确的问题；
2）亦或是浏览器准备将下一个回调推入队列，但前一个定时器的回调还处在队列中，新的回调不会被添加，而是直接跳过，这样就会造成丢帧的情况出现；

因此考虑使用 setTimeout 来模拟实现 setInterval（可以完美解决以上两个问题）：
1）setTimeout 在前一个定时器执行完前，不会向队列中推入新的定时器；
2）保证定时器执行间隔；

  • setTimeout 模拟实现 setInterval
  • 反过来也可以使用 setInterval 模拟实现 setTimeout
