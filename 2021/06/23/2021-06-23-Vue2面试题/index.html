记录一些 Vue2 相关的面试题

1. v-for 加 key 的原因

v-for 加 key 主要是为了更高效的更新虚拟 DOM，当使用 v-for 进行列表渲染时，随着数据量越来越大，我们对列表进行修改之后重新渲染视图会十分影响性能。一旦加上 key 值之后，列表修改时会根据之前的 key 值判断是否需要更新数据，如果需要则重新渲染，否则就复用先前的数据，避免了无效的更新操作影响性能。

虚拟 DOM 和真实 DOM 的区别

顾名思义，虚拟 DOM 不是真正的 DOM，它是通过 js 对象对真实 DOM 进行的模拟，因此修改虚拟 DOM 不会引起重绘和重排。等到操作完虚拟 DOM 后再与真实 DOM 进行比较，只修改有差异的部分，进行局部更新操作，这样能大大减少重绘重排带来的性能问题。

虚拟 DOM 的主要工作流程：

  1. 用 JS 对象模拟真实 DOM 树
  2. 比较两颗虚拟 DOM 树的差异
  3. 将差异应用的真正的 DOM 树上

2. 计算属性和 watch 的区别

  • computed 计算属性
    当一个数据属性在其依赖的数据属性发生变化时也要发生变化，这种情况下比较适合使用计算属性，因为 computed 是带缓存的，只有其引用的响应式数据发生变化时才会引起重新计算，否则使用上一次的缓存值。

  • watch
    大部分情况下都可以计算属性来代替 watch ，它适用于侦听一个数据的变化情况，当其发生变化时来对其他与之相关的数据进行处理（适用于执行异步操作或者处理开销较大的情况），watch 没有缓存，它会直接执行。

3. v-for 和 v-if 为什么不推荐在同一元素上使用

Vue 官方文档中提到，v-for 的优先级要高于 v-if，这意味着 v-if 将分别重复运行于每个 v-for 循环中，这对只渲染部分列表项的节点十分有用，但当我们需要有条件的跳过循环时，就应该将 v-if 置于外层元素上。

4. Vue 父子组件生命周期执行顺序

Vue 父子组件生命周期钩子函数遵循这样的规则：从外到内再从内到外。

  • 挂载阶段：父 beforeCreate -> 父 created -> 父 beforeMount -> 子 beforeCreate -> 子 created -> 子 beforeMount -> 子 mounted -> 父 mounted，子组件挂载完成后父组件才能完成挂载。
  • 更新阶段：父 beforeUpdate -> 子 beforeUpdate -> 子 updated -> 父 updated
  • 销毁阶段：父 beforeDestroy -> 子 beforeDestroy -> 子 destroyed -> 父 destroyed

5. 谈一谈 nextTick

nextTick 是 Vue 提供的一个全局 API，由于 vue 的异步更新策略导致我们对数据的修改不会立刻体现在 dom 变化上，如果想要立即获取更新后的 dom 状态，就需要使用 nextTick。

Vue 在更新 DOM 时是异步执行的，只要侦听到数据变化，Vue 将开启一个队列，并缓冲在同一事件循环中发生的所有数据变更。如果同一个 watcher 被多次触发，只会被推入到队列中一次，这对缓冲时去除重复数据以及避免不必要的计算和 DOM 操作非常重要。

nextTick 方法会在队列中加入一个回调函数，确保该函数在前面的 dom 操作完成后再调用，所以当我们想在修改数据后立即看到 dom 执行结果就需要用到 nextTick 方法。使用 nextTick 时传一个回调函数进去，在里面执行 dom 操作即可。

简单了解 nextTick 的实现，它会在 callbacks 里面加入传入的函数，然后用 timerFunc 异步方式调用它们，首选的异步方式会是 Promise。这也是为什么可以在 nextTick 中看到 dom 操作结果。

6. v-if 和 v-show 的区别

  1. 本质区别
  • v-if 是真正的条件渲染，因为它会确保在切换过程中条件块内的事件监听器和子组件适当地被销毁和重建。通过动态的向 DOM 树中增加或删除 DOM 节点来控制元素的显示和隐藏。v-if 是惰性的：如果在初始渲染时条件为假，则什么也不做——直到条件第一次变为真时，才会开始渲染条件块。
  • v-show 实质上是通过修改元素行内样式的 display 属性来控制元素的显示和隐藏，其中display:none为隐藏，display:""为显示。v-show 不管初始条件是什么，元素总是会被渲染，并且只是简单地基于 CSS 进行切换。
  1. 性能区别
  • v-if 每次切换都会修改虚拟 DOM 重新渲染，切换开销较大，适用于切换不频繁的场景。
  • v-show 除了初始渲染开销较大，后续的切换都只是对元素的 display 属性进行修改，切换开销较小，适用于切换频繁的场景。

7. Vue 组件间通信方式

  1. props/$emit
    （1）父组件向子组件传值
    父组件通过子组件标签中的属性传递数据，子组件在props中进行接收
    （2）子组件向父组件传值
    子组件通过自定义事件向父组件发送数据
  2. $children / $parent
  3. provide/inject
    简单来说就是父组件中通过 provide 来提供变量, 然后在子组件中通过 inject 来注入变量。不论子组件嵌套有多深, 只要调用了 inject 就可以注入 provide 中的数据。
  4. ref / refs
    将 ref 用在子组件上，引用指向组件实例，就可以通过实例直接调用子组件的方法或访问数据。
  5. eventBus
    通过创建一个空的 Vue 实例作为中央事件总线（事件中心），用它来触发事件和监听事件，可以实现任何组件间的通信，包括父子、兄弟、跨级，但当项目规模较大时就容易难以维护。
  6. Vuex
    Vuex 是 Vue 专用的状态库，用于管理全局的状态，适用于有较多全局状态需要管理的场景。
  7. localStorage / sessionStorage
    使用这两个 API 可以存储数据进行通信，使用简单，但是容易造成数据混乱，不易维护。
    localStorage / sessionStorage 可以结合 vuex, 实现数据的持久保存，同时解决数据和状态混乱问题。
  8. $attrs/$listeners
    可以进行父子通信和跨级通信。

具体实现及更多方式请参考

使用场景

  • 父子组件通信: props/$emit; $parent / $children; provide / inject ; ref ; $attrs / $listeners
  • 兄弟组件通信: eventBus ; vuex
  • 跨级通信: eventBus；Vuex；provide / inject 、$attrs / $listeners

9. scoped 的原理

当 <style> 标签使用 scoped 属性时，其中的 CSS 规则 只作用于当前组件中的元素。这类似于 Shadow DOM 中的样式封装，它通过 PostCSS（一个使用 js 插件来转换 CSS 的工具） 来实现。PostCSS 官网

  • 原理：在 DOM 节点以及 CSS 选择器上加唯一不重复的标记，以保证样式唯一。
    Vue 具体的做法是，为当前组件中的每个 DOM 节点都添加一个自定义属性（各组件唯一不重复），并在相应的 CSS 规则块上添加一个属性选择器进行匹配，这样就保证了样式的私有化。需要注意的是，当在组件中引用其他组件时，用于标记的自定义属性只会被添加到子组件的第一层标签中，这样设计是为了让父组件可以从布局的角度出发，调整其子组件根元素的样式。
