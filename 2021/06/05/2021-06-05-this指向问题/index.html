this指向问题
Eli Lv4
  2021-06-05 20:09:54 2021-06-05 20:09  
  • JS 
 
  • JS 

this 即函数上下文，指的是与函数调用相关联的对象。
函数的定义方式、调用方式决定了 this 的取值，然而 this 的指向也可以发生改变。

一、this 的取值

1. 函数作为函数调用（普通的函数调用）

在非严格模式下，this 指向全局 window 对象。
在严格模式下，this 指向 undefined。

2. 函数作为方法调用

this 通常指向调用该方法的对象

3. 函数作为构造函数调用

this 指向新创建的的对象

4. 使用 apply()、call()方法调用函数

this 指向 apply()、call()方法的第一个参数

apply()和 call()都可以为函数指定上下文对象，不同之处在于它们给回调函数传递参数的形式，前者传递一个参数数组，而后者传递的是参数列表。

二、改变 this 指向

除了 apply()和 call()方法，javascript 还提供了另外两种修改函数上下文的方式。

1. 箭头函数

除了可以简化函数的定义，使用箭头函数还可以绕过函数上下文。由于箭头函数没有隐式传递 this 参数，因此它并不自带函数上下文，而是从定义时所在上下文处继承，换句话说，箭头函数的 this 与声明所在的上下文的函数上下文相同。

我们需要注意箭头函数可能带来的 bug，当箭头函数定义在对象字面量中时，this 会指向全局 window 对象（严格模式下指向undefined）；在 Vue 的 watch 监听器中书写箭头函数会导致 this 指向的不是当前 vue 组件。

2. bind()方法

所有的函数实际上都是 Function 构造函数的实例，它能调用从 Function 原型继承 bind()方法，其作用是创建一个和原函数完全一致的新函数，并将其绑定到指定的对象上，这样就修改了函数的 this 指向。
