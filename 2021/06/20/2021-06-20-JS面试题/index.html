汇总一些 JS 面试题

1. var、let、const 的区别

  1. 作用域范围
    • var 声明的变量属于函数级作用域；
    • let、const 关键字声明的变量属于块级作用域；
      块级作用域解决了两个问题：一是内部变量可能覆盖外部变量，二是 for 循环中的计数变量能被外部访问。
  1. 变量提升
  • var 声明的变量在程序执行时会被自动提升到函数作用域的顶部；
  • let 和 const 声明的变量则不会在作用域中被提升；
    变量提升表现如下：
  1. 暂时性死区
  • 在使用 let、const 声明变量之前，该变量都是不可用的，这在语法上称为暂时性死区；
  • 由于变量提升的存在，所以 var 声明的变量不存在暂时性死区；

  1. 全局属性
    浏览器的全局对象是 window，Node 的全局对象是 global。var 声明的变量是全局变量，而且会成为 window 对象的属性，let 和 const 声明的变量则不会。

  2. 重复声明

  • var 可以在同一作用域内重复声明同一个变量，后面的声明会覆盖前面的；
  • const、let 则不能在同一块级作用域内重复声明同一个变量；

  1. 初始值设置
    const 声明的变量必须进行初始化，否则会报错；
    let、var 声明的变量可以不初始化，为 undefined 未定义；

  2. 值能否修改
    var 和 let 声明的变量的值可以修改，而 const 声明的变量的值是不可修改的。而对于 const 声明的引用类型，虽然不能再将变量的指向修改，但可以修改引用类型内部的结构。原因如下：

javascrit 中有两种内存结构，栈内存和堆内存，前者大小固定，用于存储各种基本类型的变量，以及引用类型的指针；后者则是一块大小不固定的内存空间，用于存储引用类型等复杂结构，为引用类型声明的变量实际上存储的是指向某块堆内存的指针。

注意事项

  • let 和 const 的出现解决了块级作用域的问题，以后应尽量避免使用 var；
  • 在非严格模式下未用关键字声明的变量将变为全局变量，在严格模式下会报错；
  • 需要一次声明多个变量，在一条语句中用逗号分隔即可，如let a=1，b=2；

2. null 和 undefined 的区别

  • null 表示一个变量被定义了，定义为空值 null；
  • undefined 表示变量不存在定义，或者是被声明了但没有进行初始化；

因此设置一个值为 null 是合理的，但设置一个值为 undefined 是不合理的。

在 JSON 规范中就不允许存在 undefined 类型，但存在表示空值的 null 。在一些使用广泛的库（如 jQuery）中的深度拷贝函数会忽略 undefined 而不会忽略 null ，也是针对上述语义的理解。

JS 中同时存在 undefined 和 null 是合理的

在 Java 中不存在 undefined 是合理的：Java 是一门静态类型语言，不可能存在一个未定义的成员（不存在的话直接就编译失败了），所以只用 null 来表示语义上的空值。而 JavaScript 作为一门动态类型语言，成员除了可能表现为空值外，还有可能根本就不存在（是否存在只在运行期才知道），所以就需要一个值来表示对某成员的 getter 是取不到值的。至于 dart 不存在 null, 可能是 dart 像 java 一样是一个静态类型语言（或者说是一个有编译期静态类型检查的语言，在运行期并不进行类型检查），所以不需要设置一个 undefined 这样的类型。

typeof null === “object” 不代表 null 是空对象

不能因为 typeof null === “object” 而认为 null 代表空对象，简单的反例：在强类型数据交换协议 odata（http://www.odata.org/）的 JSON 格式中，即使一个成员已经定义为特定类型（比如 string），也可以设置其值为 null 来表示这个值是空值。而 typeof null === “object” 像是一个设计失误，所以在 harmony 中有提议将这个返回值修正为 null :http://wiki.ecmascript.org/doku.php?id=harmony:typeof_null。但是由于该提议会造成大量旧 Javascript 脚本出现问题，因此就被否决了。

4. for…in 和 for…of 两种遍历方式的区别

  • for…in

一般用于对象的遍历，它以任意顺序遍历一个对象的属性名。但不推荐用来遍历数组，Array 作为 js 内置对象，虽然能够通过下标访问其中的元素，但它本质上还是一个对象，使用 for…in 可能会造成意想不到的问题。

for…in 会遍历数组对象中所有的属性，甚至是原本不属于这个数组对象的属性，为了避免这一问题，推荐使用 for…of 或者 forEach 进行数组遍历。

  • for…of

用于遍历可迭代对象（即具有 iterator 接口的对象，如Array，Map，Set，String，TypedArray，arguments 对象等），它按照索引顺序遍历一个可迭代对象的属性值，并且只遍历可迭代对象本身具有的属性。

5. 如何使用 for..of 遍历对象

for…of 允许遍历一个含有 iterator 接口的数据结构（数组、对象等）并且返回各项的值，普通的对象用 for..of 遍历会报错。

  1. 遍历类数组对象
    将类数组对象转化数组，再使用 for..of 遍历即可。
  1. 遍历非类数组对象
    只需要给对象添加一个[Symbol.iterator]特性，并指向一个迭代器即可。

6. requestAnimationFrame()

window.requestAnimationFrame()跟 setTimeout()类似，都用于推迟某个函数的执行，它无需指定延迟时间，一般用于操作动画。它接收一个函数作为回调，在浏览器下一次重绘之前执行回调函数来更新动画，这样能避免因动画频繁更新引发重绘造成帧率过高导致的性能问题。

刷新率和帧率

  • 刷新率
    刷新率又叫屏幕刷新率，是指显示设备一秒钟刷新图像的次数，单位为 HZ。60HZ 刷新率表示屏幕一秒钟刷新 60 次图像，也就是一秒钟最多能显示 60 帧图像。

  • 帧率
    帧率是指显示设备一秒钟展示的帧数，单位为 FPS。只有刷新率大于等于帧数时，显示设备才能显示出所有的帧。

一般浏览器帧率在 60FPS，也就是大约每 16.67ms 重绘一次，这意味着requestAnimationFrame()传递的回调函数通常每秒执行 60 次，但在遵循 W3C 标准的浏览器中，回调函数的执行次数一般与浏览器屏幕刷新次数相匹配。为了提高性能和电池寿命，在大多数浏览器里当 requestAnimationFrame() 运行在后台 Tab 标签页或者隐藏的 iframe 里时，会被暂停调用。

用法
回调函数 callback 有一个 DOMHighResTimeStamp 参数，该参数与 performance.now()的返回值相同，表示 requestAnimationFrame() 开始执行回调函数的时刻。
requestAnimationFrame() 返回一个 long 整数，类似于给 setTimeout 加的标识，可传递给 window.cancelAnimationFrame()以取消回调函数。

7. setTimeout 和 setInterval

根据事件循化机制的相关知识，延迟队列中 setTimeout 和 setInterval 的回调函数要等到任务队列中的任务执行完毕后才会执行，因此定时器的延迟时间是不准确的。

setTimeout

  • 未激活的页面 setTimeout 执行的最小时间间隔为 1000 毫秒
    处于后台运行状态的 tab 页，setTimeout 执行的最小时间间隔为 1000 毫秒，这样做是为了节约性能以及延长电池使用寿命。

  • 延迟执行时间有最大值
    Chrome、Safari、Firefox 都是以 32 bit 来存储延迟时间的，32bit 能表示的最大数为 2147483647，这意味着 setTimeout 设置的延迟时间大于 2147483647 毫秒（约 24.8 天）时就会溢出，相当于延时值被设置为 0 ，这时定时器会被立即执行。

  • setTimeout 存在嵌套调用时系统会设置最短时间间隔为 4 毫秒
    在 Chrome 中，定时器被嵌套调用 5 次以上，系统会判断该定时器被阻塞了，此时的时间间隔最小为 4 毫秒。

setInterval

  • 未激活的页面 setInterval 执行的最小时间间隔也为 1000 毫秒
  • 延迟执行时间也有最大值，和 setTimeout 一致
