JS数据类型判断方式
Eli Lv4
  2021-06-11 11:16:12 2021-06-11 11:16  
  • JS 
 
  • JS 

js 是弱类型语言，在声明变量时无需声明变量的类型，但很多时候我们需要对数据进行类型判断。

  1. typeof 操作符

typeof 操作符可以判断 js 中的几种基本数据类型(Number，String，Undefined，Boolean，Symbol)以及函数。对于 Null 型数据、数组、对象而言，使用 typeof 会返回 object。

  1. Object.prototype.toString.call()

使用 Object.prototype 上的 toString()方法判断数据类型，就比 typeof 操作符要准确的多，但它仍然不能判断某个对象实例的类型。

这里为什不直接使用 obj.toString() ，而是使用 Object.prototype.toString.call() ？实际上， toString() 作为 Object 的原型方法，是可以重写的。以数组为例，Array 等其他引用类型作为 Object 的实例，都重写了 toString()。根据原型链的原理，数组实例调用 toString 方法 时，调用的是重写之后的 toString 方法（返回数组元素组成的字符串），而不会去调用 Object 原型上的 toString 方法（返回数据类型），因此想要得到数据类型应该调用 Object 原型上的 toString 方法。

如果使用 delete 操作符将数组的 toString 方法删除，会发生什么呢？

删除 Array 的 toString 方法后，再采用 arr.toString()调用 toString 方法，不再有屏蔽 Object 原型方法的实例方法，因此沿着原型链，arr 最后调用了 Object 的 toString 方法，返回了和 Object.prototype.toString.call(arr)相同的结果。

  1. instanceof 操作符

instanceof 操作符用于检测构造函数的 prototype 是否出现在某个实例对象的原型链上，可用于判断对象实例的类型。

  1. constructor 构造函数

每个对象都会从它的原型上继承一个 constructor 属性，此属性的值是对构造函数本身的引用，而不是一个简单的字符串，通过访问该属性我们可以判断数据类型。

然而这种方式并不安全，因为 constructor 是可以改变的（原始类型的该属性是只读的，因为原始类型通过只读的原始构造函数所创建），实际生产环境中不要随意修改构造函数，否则可能会产生意想不到的 bug。
