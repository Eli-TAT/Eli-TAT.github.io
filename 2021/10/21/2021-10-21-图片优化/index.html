汇总一些开发中图片优化的方案

图片是网页中重要的表现元素，由于存在网络状况不佳、需要加载的图片资源较多、体积较大等问题，为了优化用户体验，往往需要对项目中的图片进行优化。考虑以下两种情况：

  1. 图片过多
  • 雪碧图
    对于项目中的一些小图标，如果一个一个加载会进行多次 http 请求，即使有 http 长连接，来回请求也势必会造成时间的浪费。因此可以将项目中的一些彩色小图标放入一张图中，使用时再通过 css 的background-position属性进行截取。

雪碧图在线生成工具：https://www.toptal.com/developers/css/sprite-generator、http://alloyteam.github.io/gopng/

  • base64 图片编码
    这种方式只适合对一些小的图片进行编码，因为对图片进行 base64 编码反而会增加图片体积。图片经过 base64 编码后得到的值可以直接应用于 img 标签的 src 属性，因而可以达到减少图片数量的目的。
    在线图片 base64 编码

  • iconfont 等字体库
    对于项目中的一些纯色小图标（多数为灰色，如小箭头、放大镜等），可以直接在项目中引入 iconfont，它将小图标封装成了一个字体文件，使用十分方便。iconfont 用法、iconfont 官网

  • css 代替图片
    适用于图案较为简单的场景，一般可以使用 css 绘制一些简单的图案，同时可以使用 before 或者 after 伪元素来丰富图案的复杂度。优势：实现简单，也可以实现简单的动效。劣势：受限于 css 的兼容性，绘制复杂图案困难。

  • canvas 代替图片
    适用于需要高性能的图片或动画，适用 html5 的 canvas 元素绘制创建图片。优势：使用 canvas 绘制 2D 图形，页面渲染性能比较高，受图形复杂度影响小，性能只受图形分辨率的影响，画出来的图形可以直接保存为 png 或者 jpg 个数，适合于画光栅图像或者不规则图形。劣势：没有 dom 操作，必须依赖定时器，文字渲染性能差，不能添加描述(如 title 属性)，兼容性限制。

  • svg-sprit
    顾名思义，svg-sprit 使用了 SVG 格式的图片，具有 SVG 的优点。使用它的好处在于，不用再发送 woff|eot|ttf|这些很多个字体库请求，所有的 svg 都可以内联在 html 内。
    如何在 vue 中使用 svg-sprit

  • 懒加载
    针对页面中图片过多，单无法减少的情况（如淘宝网），可以考虑懒加载，即对于视窗之外的图片，放到首屏加载完毕后再有需要的进行加载。这样既能优化首屏渲染，也能优化网页性能。可以使用现成的懒加载库，也可以自行封装。懒加载原理以及实现

  1. 图片过大

  • 选择合适的图片格式
    webP 格式的图片体积极小，但是目前浏览器支持度不够。图片格式在线转换

  • 压缩图片
    在保证分辨率的情况下，通过压缩算法对图片进行压缩，减小图片体积。（原理：减少图片中的颜色数量，去除无用的数据）
    在线图片压缩网站

  • 服务端 gzip 压缩
    gzip 是一种能够压缩多种文件的压缩手段，只需在服务器中进行相应的配置即可（如 nginx）

  • 服务端对图片进行裁剪

  • 根据设备加载合适的图片
    在移动端展示分辨率较高的图片实际上会造成流量的浪费，没有必要。因此可以针对不同设备制作相应的图片，使用 img 标签的srcset属性，根据当前设备的分辨率加载合适的图片。

  • 模糊加载
    当图片过大时，可以使用这种方式提高用户体验。实现方法如下：
    （1）页面开始加载，先为图片展示区域加载一个占位符（占位符可以是跟图片颜色相近的背景色、也可以是一个缩略图）
    （2）占位符加载的同时绘制一个 canvas，将图片的数据传递给自定义的 blur 函数，设置透明度，同时开始请求大图
    （3）大图加载完毕后，canvas 会隐藏掉，可以通过 transition 处理加载过程，制造出一个模糊加载的效果
    知乎、Medium 就用到了这种方式，具体实现实现可参考：知乎、Medium
