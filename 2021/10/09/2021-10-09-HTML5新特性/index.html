汇总一些 HTML5 新特性及用法

1. 语义化

语义化是 HTML5 新特性，通俗来讲就是用合理的标签展示内容，如用 header 标识页眉、nav 标识导航栏等。

语义化优点

  • 易于用户阅读，样式丢失的时候能让页面呈现清晰的结构
  • 有利于 SEO，搜索引擎根据标签来确定上下文和各个关键字的权重
  • 方便其他设备解析，如盲人阅读器根据语义渲染网页
  • 有利于开发和维护，语义化更具可读性，代码更好维护，与 CSS3 关系更和谐

2. 地理定位

地理定位（Geolocation）是 HTML5 提供的一个新功能，用于获得用户的地理位置。鉴于该特性涉及到用户隐私，除非用户同意开启，否则地理定位功能是不可用的。除了主流浏览器支持该 API，对于拥有 GPS 的设备（比如 iPhone），地理定位会更加精确。

  • getCurrentPosition() 方法获取当前位置
    getCurrentPosition()接收两个函数作为参数，第一个参数用于获取和处理位置信息，执行成功后会得到一个 coordinates 对象，记录了当前定位的经纬度信息；第二个函参数用于错误处理。

  • 如何在地图中显示当前位置
    如果需要在地图中显示当前位置，则需要访问可使用经纬度的地图服务，比如谷歌地图或百度地图，直接利用 getCurrentPosition()获取到的位置坐标在地图服务中进行定位即可。

  • 给定位置信息时地理定位的作用
    更新本地信息；显示用户周围的兴趣点；实现交互式导航系统 (GPS)

  • Geolocation 对象的其他方法
    watchPosition()会返回用户的当前位置，并继续更新用户移动时的位置（需要一台精确的 GPS 设备进行测试，例如 iPhone）。
    clearWatch()停止 watchPosition() 方法

3. Canvas 画布

canvas 是 HTML5 提供的一个新标签，通过 JavaScript 来操控画布，就可以在网页上绘制图像。画布实际上就是一个矩形区域，我们可以通过各种绘制路径、矩形、圆形、字符以及添加图像的方法来在画布上绘制图像。

使用步骤

  1. 创建 Canvas 元素
    在 HTML5 文档中添加 canvas 元素，并规定元素的 id、宽度和高度。

  2. 通过 JavaScript 绘制
    canvas 元素本身是没有绘图能力的，所有的绘制工作必须使用 JavaScript 完成。
    （1）通过 id 获取到 canvas 元素
    （2）使用 getContext(“2d”)创建 一个 context 对象（context 对象是内建的 HTML5 对象，拥有多种绘制路径、矩形、圆形、字符以及添加图像的方法）
    （3）使用各种 API 绘制图形

注意事项

  • 当 canvas 画布元素的 width 和 height 属性被修改时，画布上的内容会被全部擦除。width 和 height 的默认值都为 300。
  • 在 Vue 中使用 canvas 时，要注意在 mounted()阶段之前还获取不到画布元素的 DOM 节点，因此无法进行绘制操作。

其他应用

  1. 在画布上放置图片
    使用 context 对象上的 drawImage()方法即可，但是要注意图片的来源，加载其他域的图片会受到浏览器同源限制。
  • 本地图片
  • 来自其它域的图片
  1. 如何将画布转换为图片并下载到本地
    1）使用toDataURL()方法生成一个url
    2）使用 createElement()创建一个 a 标签，将画布元素的 url 绑定到 a 标签的 href 属性，设置 a 标签的download属性（下载链接的路径）
    3）对 a 标签模拟点击

4. SVG

什么是 SVG
SVG 全称可伸缩矢量图，使用 XML 格式定义图形，是 W3C 的图形标准，SVG 图像在放大或改变尺寸的情况下图形质量不会有损失。HTML5 支持内联 SVG，可以将 SVG 元素直接嵌入 HTML 文档中。

SVG 的优势

  • SVG 图像可通过文本编辑器来创建和修改
  • SVG 图像可被搜索、索引、脚本化或压缩
  • SVG 图像可在任何分辨率下被高质量地打印
  • SVG 可在图像质量不下降的情况下被放大

5. 媒体元素

HTML5 支持 video 和 audio 媒体元素，可以直接在页面中插入视频和音频。

常用属性

  • controls，设置音频/视频是否显示控件
  • loop，是否循环播放
  • paused，是否暂停
  • muted，是否静音
  • autoplay，加载完毕后是否自动播放
    …

常用方法

  • play()
  • pause()
  • load()
    …
    更多属性和方法请参考

注意事项

  • 在设置 autoplay 自动播放属性时，浏览器出于对用户体验的考虑，audio 不支持加载完毕后自动播放，video 加载完毕后需要处于静音状态才能自动播放。
  • 可以设置 controls 使用视频和音频元素的默认控件；也可以使用 CSS 设计样式，封装相应的属性和方法来打造自定义的媒体元素。

6. HTML5 拖放

拖放是 HTML5 提供的新特性：任何元素都是可拖放的。

简单使用步骤

  1. 将元素设置为可拖放
    只需将draggable 属性设置为 true，例如
  2. 处理被拖放元素
    dragstart 事件中规定拖动什么数据，dataTransfer.setData() 用于设置被拖动数据的数据类型和值。
  3. 处理目标元素
    dragover 事件规定被拖动的数据能够被放置到何处。默认地，数据/元素无法被放置到其他元素中，因此需要阻止元素的这种默认行为（使用 event.preventDefault()）。
  4. 进行放置
    当放开被拖数据时，会发生 drop 事件。在这里也需要阻止元素的默认行为，通过 dataTransfer.getData()可以获取到被拖动数据。

使用实例可参考

涉及到的事件

  • dragstart：事件主体是被拖放元素，在开始拖放被拖放元素时触发。
  • darg：事件主体是被拖放元素，在正在拖放被拖放元素时触发。
  • dragenter：事件主体是目标元素，在被拖放元素进入某元素时触发。
  • dragover：事件主体是目标元素，在被拖放在某元素内移动时触发。
  • dragleave：事件主体是目标元素，在被拖放元素移出目标元素时触发。
  • drop：事件主体是目标元素，在目标元素完全接受被拖放元素时触发。
  • dragend：事件主体是被拖放元素，在整个拖放操作结束时触发。

7. Web Worker

通过使用 Web Worker，Web 应用可以在独立于主线程的后台线程中运行脚本。好处是可以在独立线程中执行费时的处理任务，从而允许主线程（通常是 UI 线程）不会因此被阻塞/放慢。Worker 指的是一种可由脚本创建的后台任务，任务执行中可以向其创建者收发信息。
具体用法参考 MDN

主线程和 worker 线程相互之间使用 postMessage() 方法来发送信息, 并且通过 onmessage 这个 event handler 来接收信息。数据的交互方式为传递副本，而不是直接共享数据。

除了专用 worker 之外，还有一些其他种类的 worker ：

  • Shared Workers 可被不同的窗体的多个脚本运行，例如 IFrames 等，只要这些 workers 处于同一主域。
  • Service Workers 一般作为 web 应用程序、浏览器和网络之间的代理服务。旨在创建有效的离线体验，拦截网络请求，以及根据网络是否可用采取合适的行动，更新驻留在服务器上的资源。他们还将允许访问推送通知和后台同步 API。一般可以用来实现缓存功能。

8. 新的表单特性

表单类型
email ：能够验证当前输入的邮箱地址是否合法
url ： 验证 URL
number ： 只能输入数字，其他输入不了，而且自带上下增大减小箭头，max 属性可以设置为最大值，min 可以设置为最小值，value 为默认值。
search ： 输入框后面提供一个清除内容的按钮。
range ： 提供一个范围，可以设置 max 和 min 以及 value，其中 value 属性可以设置为默认值
color ： 提供了一个颜色拾取器
time ： 时分秒
data ： 日期选择年月日
datatime ： 时间和日期(目前只有 Safari 支持)
datatime-local ：日期时间控件
week ：周控件
month：月控件

表单属性
placeholder：提示信息
autofocus：自动获取焦点
autocomplete=“on” 或者 autocomplete=“off” 使用该属性需要有两个前提：表单必须提交过；必须有 name 属性。
required：要求输入框不能为空，必须有内容才能够提交。
pattern：里面写入正则表达式进行匹配，例如手机号 patte=”^(+86)?\d{10}$”
multiple：可以选择多个文件或者多个邮箱
form：form 表单的 ID

表单事件
oninput 当 input 输入框里的内容发生变化时触发。
oninvalid 当验证不通过时触发。

10. 进度条、度量器

progress 标签：用来表示任务的进度（IE、Safari 不支持），max 用来表示任务的进度，value 表示已完成多少
meter 属性：用来显示剩余容量（IE、Safari 不支持）

high/low：规定被视作高/低的范围
max/min：规定最大/小值
value：规定当前度量值

设置规则：min < low < high < max

11. Web 存储

localStorage - 本地存储，长期有效
sessionStorage - 会话存储，当前窗口内有效

有同源限制。要注意的是，它们只能存储字符串类型的数据，因此存储 js 对象时要使用 JSON.stringfy()进行序列化，取出时再使用 JSON.parse()解析为对象。
