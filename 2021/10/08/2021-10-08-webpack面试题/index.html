汇总一些 webpack 面试题参考

1. Loader 和 Plugin

  • Loader
    Loader 译为”加载器”。Webpack 将一切文件视为模块，但是 webpack 原生只能解析 js 文件，如果想将其他文件也打包的话，就会用到 loader。Loader 让 webpack 拥有了加载和解析非 JS 文件的能力。
    Loader 在 module.rules 中配置，也就是说它作为模块的解析规则而存在。rules 是一个数组，其中每一项都是一个对象，描述了对于什么类型的文件（test），使用什么加载(loader)和使用的参数（options）
  • Plugin
    Plugin 译为”插件”。Plugin 可以扩展 webpack 的功能，让 webpack 具有更多的灵活性。 在 Webpack 运行的生命周期中会广播出许多事件，Plugin 可以监听这些事件，在适当的时机通过 Webpack 提供的 API 改变输出结果。
    Plugin 在 plugins 中单独配置。Plugins 是一个数组，每一项是一个 plugin 的实例，参数都通过构造函数传入。

2. 常用的 loader 和 plugin

  1. loader

    • file-loader：把文件输出到一个文件夹中，在代码中通过相对 URL 去引用输出的文件
    • url-loader：和 file-loader 类似，但是能在文件很小的情况下以 base64 的方式把文件内容注入到代码中去
    • source-map-loader：加载额外的 Source Map 文件，以方便断点调试
    • image-loader：加载并且压缩图片文件
    • babel-loader：把 ES6 转换成 ES5
    • css-loader：加载 CSS，支持模块化、压缩、文件导入等特性
    • style-loader：把 CSS 代码注入到 JavaScript 中，通过 DOM 操作去加载 CSS。
    • eslint-loader：通过 ESLint 检查 JavaScript 代码
    • svg-sprite-loader：将svg文件打包成svg-sprite

  2. plugin

    • SplitChunksPlugin：代码分割插件，十分常用；参考
    • BundleAnalyzerPlugin：生成当前项目的文件依赖图，为打包优化提供分析依据（vue-cli 中自带）
    • HappyPack：如果项目过大，打包时间过长，可以使用HappyPack 插件，开启多进程打包
    • define-plugin：定义环境变量
    • commons-chunk-plugin：提取公共代码
    • uglifyjs-webpack-plugin：通过 UglifyES 压缩 ES6 代码

Sourcemap 是什么，其作用及用法？

Sourcemap 是什么？Sourcemap 的作用及用法概括

Sourcemap 实质上是一个信息文件，用来记录项目源代码和打包后代码间的映射关系，主要用于方便 debug。由于打包后的项目代码经过了压缩、混淆、合并等操作，与源代码不再一致，这时如果要对线上问题进行 debug 就十分困难。sourceMap 的作用就是能够让浏览器的调试面版将打包后的代码映射到源码文件当中，开发者可以直接在源码文件中 debug，方便调试。

使用
Sourcemap 的种类有很多，可以根据实际运行环境进行设置。例如在 webpack 中通过devtool进行设置，有以下选项可供使用：

  • cheap-module-source-map，开发环境中可以使用
  • none，建议生产环境下使用，这样不会暴露源代码
  • nosources-source-map，既可以定位源代码位置，又不暴露源代码
