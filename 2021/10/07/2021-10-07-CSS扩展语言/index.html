记录一下几种常见的 CSS 扩展语言的特点与区别。
关于 Sass 与 Less 的区别可以看
在vue中使用sass存在的坑

常见的 CSS 扩展语言有 Sass/Scss、Less、Stylus 等，它们是 CSS 的超集，提供了一些 CSS 不具有的功能，特别是加入了一些编程特性（例如变量、循环、函数等），这使得开发者可以使用常用的编程思想以及强大的语法来优雅地编写 CSS。然而浏览器只能识别普通的 CSS，因此如果想要在项目中使用这些 CSS 扩展语言，就需要通过CSS 预编译器将 CSS 扩展语言编写的 CSS 编译为浏览器能够识别的 CSS。

1. Sass/Scss

Sass 官网地址

Sass 与 Scss 其实师出同门，Scss 实际上就是 Sass3 之后对 Sass 的改良版本，两者的最大区别在于：Sass 使用缩排语法，Scss 则用{}替代了 Sass 中的缩进，这使得在 Scss 中可以使用最普通的 CSS 代码块。

编译方式
Sass 需要在服务端进行处理才能编译为普通的 CSS，以前使用 Ruby，现在是 Dart-Sass 或 Node-Sass，甚至可以进行在线转化。Sass 在线转换

特点

  • 使用变量
    使用$符号声明和引用变量。
  • 嵌套规则
    1）CSS 规则块可以嵌套，以此减少选择器的重复书写。
    2）在解开一个嵌套的 CSS 规则块时 Sass 会把父选择器通过一个空格连接到子选择器前，当书写伪类时这种默认规则就会出现一定的问题，这时可以使用一个特殊的 sass 选择器&（父选择器）。
    3）属性也可以进行嵌套，例如 border 属性。
    …
  • SASS 文件导入（@import）
    与 CSS 的@import 规则不同的是，sass 的@import 规则会在生成 css 文件时就把相关文件导入进来，无需发起额外的下载请求。而且所有在被导入文件中定义的变量和混合器均可在导入文件中使用。
    …
  • 静默注释
    sass 提供了一种不同于 css 标准注释格式/_ … _/的注释语法，即静默注释，其内容不会出现在生成的 css 文件中。
  • 混合器
    混合器使用@mixin 标识符定义，它用于定义一段可重用的样式，然后就可以在样式表中通过@include 来调用，它会把混合器中的所有样式提取出来放在@include 被调用的地方。
    …
  • 选择器继承
    选择器继承是指一个选择器可以继承另一个选择器定义的所有样式，通过@extend 实现。
    …

2. Less

Less 官网地址
编译方式
Less 需要引入专门的 less.js 来将 Less 转换为普通的 CSS，也可以在服务端将 Less 文件编译成 css 文件，输出 CSS 文件到项目目录，有 npm less, Less.app、SimpleLess、CodeKit.app 等工具，甚至也可以进行在线编译

特点

  • 使用变量
    使用@符号声明和引用变量

3. Stylus

Stylus 官网地址
Stylus 最直观的特点就是使用缩排语法，规则块中的某些符号可有可无（如花括号、分号等）。

特点

  • 使用变量
    无需特殊符号就能声明和引用变量
