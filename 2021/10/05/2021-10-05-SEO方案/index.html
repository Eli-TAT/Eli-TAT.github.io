SEO方案

SEO 即搜索引擎优化，有利于我们的网页被搜索引擎爬虫获取到，在搜索结果中排在较前的位置。

1. 多页面应用

对于传统的多页面应用来说，可以使用如下方式：

  • 恰当的设置网页的tittle，meta 标签中的 keywords、description 属性；
  • 使用 HTML5 语义化标签，使得搜索引擎爬虫能够清晰的理解页面结构；
  • 将网页的重要内容放在文档的前面部分，因为搜索引擎爬虫抓取的网页内容长度是会有限制的；
  • 页面中的强调内容使用strong标签进行强调，使用h1等标签来设置有层级关系的标题；
  • 网页的重要内容不要使用 js 生成，因为搜索引擎爬虫不会执行去 js 程序；
  • 所有的非装饰性图片都设置 alt 属性，作为图面加载失败时的展示内容；
  • 优化网页加载时间，因为加载时间也是搜索引擎爬虫为网页评分的一个重要参考；

2.SPA（单页面应用）

对于 SPA（单页面应用），由于其网页内容都是通过 js 脚本生成的，而搜索引擎爬虫又不会去执行 js 脚本（目前谷歌等搜索引擎已经支持爬虫程序执行同步的 js 脚本，但不支持异步的 js 脚本），爬虫程序无法获取到网页的主要内容，也就不利于进行 SEO。因此对多页面应用的优化手段也就无法直接应用到 SPA。可考虑的方案如下：

  1. SSR，即服务端渲染，指的是先在服务器端对网页进行渲染，生成 HTML 文档并写入数据，再将文档返回客户端进行视图渲染。

    • 优点
      不仅可以提高客户端网页的首屏加载速度，也有利于 SEO，因为搜索引擎爬虫获取到的网页已经渲染出了内容。
    • 缺点
      SSR 的缺点也很明显，对文档的渲染操作会增大服务器的负载，同时对开发也有一定的限制（以 Vue 为例，在 Vue 的 SSR 框架 nuxt 中，只支持 beforeCreate、created 两个生命周期）

  2. 构建时预渲染

    利用 webpack 等构建工具，针对 SPA 应用开发后只有一个 index.html 文件入口，用 puppeteer 等预渲染中间件在前端项目构建时预先获取页面数据，生成多个页面，如 about、help 、contact 等页面，优化首屏渲染与部分页面的 SEO。

    • 优点
      代码侵入性小
    • 缺点
      无法用于大量动态路径页面场景（生成的 html 页面数据大，而且页面数据会有更新。如 /article/123，文章页面）
      后台请求数据变动时前端应该同步更新版本；

  3. 服务端动态渲染（利用user-agent）
    简单来说就是服务器针对不同的用户代理，提供不同的页面，提供给浏览器端的是SPA初始的HTML，而提供给爬虫程序的则是渲染好的HTML，这样既兼顾了SPA使用灵活性的同时也兼顾了SEO。具体方案可参考SPA 的 SEO 方案对比、最终实践
