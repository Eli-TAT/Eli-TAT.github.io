事件冒泡和事件捕获
Eli Lv4
  2021-07-31 11:20:29 2021-07-31 11:20  
  • JS 
 
  • JS 

为了解决页面中事件流相关的问题，微软和网景公司分别提出了事件冒泡和事件捕获的概念

事件流

简单来说，事件流描述了用户与浏览器交互过程中事件发生的顺序。

事件冒泡

指事件按照从最精确的事件目标到最不精确的事件目标(document 对象)的顺序触发，即从内到外。现代浏览器中事件触发顺序默认都是事件冒泡。

事件捕获

事件按照从最不精确的事件目标到最精确的事件目标的顺序触发，与事件冒泡相反，即从外到内。通过设置 element.addEventListener(event, function, useCapture) 的第三个参数为 true 启用事件捕获。

阻止事件冒泡

  1. 通过 event.stopPropgation()

在最内层元素的事件方法中写入即可。（PS：event.cancelBubble=true 相当于该方法的一个曾用名，但 Firefox 不支持）。

  1. 通过 return false

在 javascript 中 return false 只会阻止默认事件，而在 jQuery 中既能阻止默认事件又能阻止事件冒泡。（PS：event.preventDefault()用于阻止默认事件）。

事件委托

简单来说，事件委托就是将事件监听添加到上级元素，通过冒泡机制实现，可以优化性能，比如为长列表绑定事件。

假如需要通过点击删除列表中的某个元素，如果给每个 li 标签都绑定一个点击事件，代码既臃肿又影响性能。因此可以使用事件委托，只给父级元素 ul 绑定一个点击事件，通过冒泡机制，点击 li 元素会触发 ul 上的事件，从而调用事件处理函数实现删除功能。
