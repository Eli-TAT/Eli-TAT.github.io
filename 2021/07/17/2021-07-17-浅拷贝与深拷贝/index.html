浅拷贝与深拷贝

深拷贝是针对引用数据类型而言的，对于基本数据类型只有浅拷贝。参考文章

两种数据类型

JavaScript 中的数据类型可分为两类，基本数据类型和引用数据类型。其中基本数据类型是按值访问的，不会影响到其他数据；而引用数据类型是按地址访问的，简单的赋值实际上只复制了地址，修改任意一个值会影响到另外一个，这就是所谓的浅复制。

深拷贝与浅拷贝

浅拷贝和深拷贝（又叫浅复制和深复制）

  • 对基本数据类型而言，浅拷贝是对值的拷贝，没有深拷贝的概念。
  • 对引用类型来说，浅拷贝是对对象地址的拷贝，并没有开辟新的栈，复制的结果是两个对象指向同一个地址，修改其中一个对象的属性，另外一个对象的属性也会改变；而深拷贝则会开辟新的栈，创建一个新的对象并为其添加和原对象相同的属性，复制出的对象和原对象不会相互影响。

深拷贝的实现

  1. 将对象转 json 字符串，再转对象，即JSON.parse(JSON.stringify(target))

    • 原理
      通过 JSON.stringify()将对象转换成 json 字符串，再用 JSON.parse()把 json 字符串解析成对象，这样会产生新的对象，而且会开辟新的栈，实现深拷贝。
    • 优点
      简单快捷
    • 缺点
      1）JSON.stringify()只能序列化对象的可枚举的自有属性，转换过程中引用类型的属性值会丢失原本的 constructor，统统变为 Object，尽管对象的 constructor 可以变更，不会带来什么特别大的影响，但习惯上还是应该让对象的 constructor 指向正确的构造函数；
      2）此外，RegExp、function 等类型无法通过这种方式进行深复制，因为它们无法正常转换为 json 格式，它们的转换结果分别为{}和 undefined；
      3）Date 类型在经历转换之后，最后复制得到的时间对象是字符串形式，而不是一个 Date 类型；
      4）无法解决对象循环引用的问题；

  2. 通过递归拷贝

  • 原理
    遍历需要拷贝的数据结构，判断其属性是否为引用类型，如果不是则直接拷贝；如果是则判断是数组还是对象，进行对应的初始化，再递归调用自身，重复上诉过程。

改进
1）复制过程中如果遇到相互引用的对象，会出现死循环的情况。可以使用一个 WeakMap 存储已经被拷贝过的对象，每一次拷贝前先查询该对象是否已经被拷贝，如果已经被拷贝则取出该对象并返回。
