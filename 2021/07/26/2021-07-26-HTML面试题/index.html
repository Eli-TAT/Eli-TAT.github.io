汇总一些 HTML 面试题

1. url、src、href 的区别

– url src href
全称 统一资源定位符 资源 超文本引用(Hypertext Reference)
作用 标识资源的地址 将资源下载并应用到文档中 建立元素与资源或锚点间的链接
常用标签 – script、img a、link

在 HTML 文档渲染过程中，img 标签和 script 标签需要先将资源下载到本地才能应用到文档中，这样会阻塞 DOM 的渲染，可以利用这一点提升网页加载速度。如：

  • 减少网络请求次数（使用雪碧图、合并 js 文件等）；
  • 提高静态资源加载速度（压缩 js 代码、使用 CDN 等）；
  • 异步加载 js 文件（使用 script 标签的 defer、async 属性）；

PS：浏览器支持并行加载静态文件（如图片、css 文件、js 文件），但是有限制，比如 Chorme 在 HTTP1.1 协议下，最多只支持 6 个文件并行加载。

2. async 和 defer 属性的区别

HTML 中 script 标签负责加载、执行 js 文件，并且是加载完成后立即执行。也就是说在 HTML 文档解析过程中，一旦碰到 script 标签，后续文档的解析要等到 js 文件加载并执行完后才会继续，这样会造成 DOM 渲染的阻塞，这时就要用到 async 和 defer 属性。

  • 将 script 标签放在文档底部
    理论上来说,将脚本文件都放在 body 闭合标签前，就能保证其他所有非脚本元素能够以最快的速度得到加载和解析，对于旧浏览器而言这是唯一的优化选择。

  • defer 属性
    js 文件的加载执行与 HTML 文档的解析并行(异步)进行，js 文件加载完成后，需等待文档全部解析完成再执行。如果加载了多段脚本，按顺序执行。

  • async 属性
    js 文件的加载执行与 HTML 文档的解析并行(异步)进行，js 文件加载完成后立即执行，不管后续文档是否全部解析完成。如果加载了多段脚本，不会按顺序执行。

3. DOCTYPE(⽂档类型) 的作⽤

DOCTYPE 是标记语言的文档类型声明，它用于告诉浏览器以什么样的方式来解析渲染文档，不同的渲染模式会影响浏览器对 CSS 甚⾄ JS 脚本的解析。它必须声明在 HTML ⽂档的第⼀⾏。文档模式首先由 IE 提出，后来其他浏览器也相继实现了文档模式的切换。

两种文档模式
可通过 document.compatMode 获取当前页面的文档模式。

  • CSS1Compat：标准模式（严格模式），浏览器使用 W3C 标准解析渲染页面，以其支持的最高标准呈现页面。
  • BackCompat：混杂模式(怪异模式)，通过省略文档开头的 DOCTYPE 开启，在这种模式下，浏览器使用自己特有的规则解析渲染页面，以一种比较宽松的向后兼容的方式进行渲染。混杂模式在不同浏览器中的差异非常大，并不合理。

4. <!Doctype html>

的作用就是让浏览器进入标准模式，使用最新的 HTML5 标准来解析渲染页面；如果不写，浏览器就会进入混杂模式。

文档中第一行的 DOCTYPE 就被称为DTD，DTD 决定了浏览器使用严格模式还是混杂模式。
而 HTML5 没有 DTD ，因此也就没有严格模式与混杂模式的区别，HTML5 有相对宽松的渲染方法，已经尽可能大的实现了向后兼容。

5. 常用的 meta 标签

meta 标签由 name 和 content 属性定义，用来描述网页文档的属性，例如网页作者、网页描述、关键词等，除了 HTTP 标准规定的一些 name，开发者也可以自定义 name。

常用 meta 标签

  • charset，用来描述 HTML 文档的编码类型
  • keywords，页面关键词
  • description，页面描述
  • refresh，页面重定向和刷新（可以定义时间间隔定期刷新）
  • viewport，适配移动端，控制视口的大小和比例

可用参数

  • width：宽度(数值或 device-width)

  • height：高度(数值或 device-height)

  • initial-scale ：初始缩放比例

  • maximum-scale ：最大缩放比例

  • minimum-scale ：最小缩放比例

  • user-scalable ：是否允许用户缩放(yes/no）

  • 搜索引擎索引方式

可用参数

  • all：文件将被检索，且页面上的链接可以被查询；
  • none：文件将不被检索，且页面上的链接不可以被查询；
  • index：文件将被检索；
  • follow：页面上的链接可以被查询；
  • noindex：文件将不被检索；
  • nofollow：页面上的链接不可以被查询。s
  • x-ua-compatible，指定浏览器渲染文档的模式（IE 专用），用于在高版本 IE 中兼容低版本 IE 特有的 API，只有 IE9 以上支持

6. 常见的行内元素、块级元素、空元素分别有哪些

  • 行内元素
    行内元素不单独成行，不能设置宽高，但是可以设置水平方向的margin和padding，内部可以包含行内元素，不能包含块级元素。
    a span img script br button input label select strong em b
  • 元素
    块级元素独占一行，可以设置宽高以及margin、padding，内部可以包含块级元素和行内元素。
    div ul ol li dl dt dd h1 h2 h3 h4 h5 h6 p
  • 空元素（也称void elements）
    只有一个标签（最好将标签闭合），用来在元素位置插入或嵌入一些内容。
    <br>、<hr>、<img>、<input>、<link>、<meta>

7. img 元素的 srcset 属性

img 的 srcset 属性用于设置不同屏幕密度下，自动加载不同质量的图片。这在响应式布局中有良好的应用，img 元素会根据设备的屏幕密度来自动加载最合适的图片，能够保证图片清晰度和加载速度。

按照上面的用法，不同的屏幕密度都要设置图片地址，目前的屏幕密度有 1x,2x,3x,4x 四种，如果每一个 img 元素都设置 4 个 URL，加载会很慢。所以就有了新的 srcset 标准。

其中 srcset 指定图片的地址和对应的图片质量。sizes 用来设置图片的尺寸零界点。对于 srcset 中的 w 单位，可以理解成图片质量。如果可视区域小于这个质量的值，就可以使用。浏览器会自动选择一个最小的可用图片。

8. Content-Type

Content-Type 用于定义网络文件的类型，决定浏览器将以什么形式读取这个文件。HTTP 请求头中的 Content-Type 指示向客户端向服务器请求什么类型的文件，HTTP 响应头中的 Content-Type 标头则告诉客户端实际返回的内容的类型。更多 Content-Type 请看

常见的媒体格式类型
text/html ： HTML 格式
text/plain ：纯文本格式
text/xml ： XML 格式
image/gif ：gif 图片格式
image/jpeg ：jpg 图片格式
image/png：png 图片格式

application 开头的媒体格式类型
application/xml： XML 数据格式
application/json： JSON 数据格式
application/octet-stream ： 二进制流数据（如常见的文件下载）
application/x-www-form-urlencoded ：form 元素中默认的 encType，form 表单数据被编码为 key/value 格式发送到服务器（表单默认的提交数据的格式）

其他
multipart/form-data：需要在表单中进行文件上传时，就需要使用该格式

9. title 与 h1 的区别、b 与 strong 的区别、i 与 em 的区别

  • strong 标签有语义，起到加重语气的效果，而 b 标签没有，它只是一个简单的字体加粗标签。它们都能将字符都设为粗体，而搜索引擎更侧重 strong 标签。
  • title 属性没有明确意义只表示是一个标题，h1 则表示层次明确的标题，对搜索引擎抓取页面信息有较大的影响
  • i 内容展示为斜体，em 表示强调的文本

10. iframe 的优点和缺点

iframe 元素用于在网页中嵌入另外一个文档。

  • 优点
    用来加载速度较慢的内容（如广告、示例等）
    可以使脚本并行下载
    可以实现跨子域通信

  • 缺点
    iframe 会阻塞主页面的 onload 事件
    无法被一些搜索引擎索识别
    会产生很多页面，不易管理

11. label 的作用

label 标签可以用于关联表单控件，当用户选择 label 标签时，浏览器会自动将焦点定位到和 label 标签关联的表单控件上。

12. 聊一聊 head 标签

head 标签用于定义文档的头部信息，是头部元素的容器，其中包含的元素具有引用js脚本、链接css样式文件、设置元信息等功能。文档的头部描述了文档的各种属性和信息，包括文档标题、在 Web 中的位置以及和其他文档的关系等。绝大多数文档头部包含的数据都不会真正作为内容显示给用户。
