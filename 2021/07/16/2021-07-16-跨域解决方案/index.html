跨域解决方案

跨域是开发中常见的问题，有多种解决方式。更多方式及具体用法参考

同源策略
浏览器的同源策略导致了跨域问题的存在，同源策略作为浏览器的一种重要的安全措施，有效防止了互联网上对浏览器潜在的恶意攻击。所谓同源，就是指URL中的协议、域名、端口号均相等。同源策略限制以下几种行为

  • Cookie、LocalStorage 和 IndexDB 无法读取
  • DOM 和 Js对象无法获得
  • AJAX 请求不能发送

跨域方案

  1. jsonp：jsonp 的原理是利用 script 标签的跨域特性，可以不受限制地从其他域中加载资源，类似的标签还有 img。要注意的是 script 标签只支持 get 请求方式。

  2. document.domain：这种方式用在主域名相同子域名不同的跨域访问中。

  3. window.name：在一个窗口(window)的生命周期内，窗口载入的所有的页面都是共享一个 window.name 的，每个页面对 window.name 都有读写的权限，window.name 是持久存在一个窗口载入过的所有页面中的，并不会因新页面的载入而重置。window.name用法参考

  4. window.postMessage：window.postMessage 是 H5 中的一个新 API，可以实现跨域访问，使用它可以向其它的 window 对象发送消息，无论该 window 对象是同源还是不同源。
    例如github 中就使用了该 API，用于实现登录后通知其他未登录页面的功能。postMessage用法参考

  5. CORS：全称跨域资源共享，CORS 的基本思想简单来说，就是使用自定义的 HTTP 头部让浏览器与服务器进行沟通，从而决定请求或响应应该成功还是应该失败。CORS用法参考

在服务端设置相应的响应头即可实现CORS

  1. WebSocket：Websocket 协议是一种建立在 TCP 协议上的通讯协议，不受同源策略限制，可以进行跨域通讯。使用JS 创建一个 Websocket 连接之后，浏览器会发起一个 HTTP 请求，取得服务器响应后，建立的连接会从 HTTP 协议切换为 WebSockt 协议（HTTP状态码为101）。
    WebSocket用法参考 、WebSocket 教程

  2. nginx代理
    浏览器存在同源限制，但服务器没有，也就不存在跨域问题。因此可以通过nginx配置一个代理服务器，将前端的请求转发给对应的后端服务器。
