解决移动端设置边框宽度为 1px，但 ios 端边框从视觉上来看较粗的问题

首先需要理解物理像素、逻辑像素、设备像素比这三个概念。

  • 物理像素(DP)
    又称称硬件像素，指的是不同设备屏幕显示的最小单位；

  • 逻辑像素：
    即 css 中设置的像素；

  • 设备像素比(DPR)
    物理像素与逻辑像素之比，代表了一个逻辑像素在渲染时对应多少个物理像素；

以 iphone6 为例，由于其设备像素比等于 2，也就是说当设置 border-width=1px 时，1px 逻辑像素对应着 2px 的物理像素，因此边框视觉上看起来会比安卓设备要粗。

解决方案

1、 媒体查询

利用媒体查询，根据设备像素比计算应该设置的边框宽度大小；

优点：简单，好理解
缺点：兼容性差，目前只有 IOS8+才支持，在 IOS7 及以下、安卓系统都显示为 0px，不推荐使用。

2、transform

为容器添加伪元素，通过 position 定位属性使其与容器重合，设置伪元素宽高为 200%、边框宽度为 1px，再使用 transform 属性的 scale()进行缩放，就能使边框在视觉上呈现 1px 的大小。

优点：灵活性强，可以实现圆角，推荐使用
缺点：为按钮添加 active 效果比较麻烦

3、 媒体查询 + transfrom

相当于是通过transform对方案1的优化版本。

除此之外，还可以通过background-image 渐变实现、box-shadow实现等方案
