低版本 IE 在事件处理上与其他遵循 W3C 标准的浏览器存在一定的差异，需要进行兼容处理。

通过在CAN I USE查阅兼容性发现，IE 从 IE9 开始在事件处理上与 W3C 标准保持一致，IE8 及以下版本则需要进行兼容处理，下面列出了 IE8 及以下版本 在事件处理上与 W3C 标准存在的差异。

  • 获取事件对象
    W3C 标准: e
    IE8 及以下: window.event

  • 获取触发事件的节点
    W3C 标准: e.target 　　　　　　　　　　　
    IE8 及以下: e.srcElement;

  • 阻止冒泡
    W3C 标准：stopPropagation()
    IE8 及以下： cancelBubble = true;

  • 阻止默认操作
    W3C 标准： preventDefault()
    IE8 及以下： returnValue = false;

  • 事件监听
    W3C 标准： document.addEventListener(event, func, useCapture)
    IE8 及以下： document.attachEvent(on+’event’, func)

兼容方案

  1. 使用 jquery1.x 或 2.x 版本进行兼容处理
  2. 根据 IE 处理事件的特性，自主封装方法进行兼容
  3. 通过 polyfill，为 IE 添加 W3C 标准的事件处理 API，可参考MDN

简单实现

下面以监听点击事件为例进行简单的兼容实现，由于本人电脑自带 IE11，故使用 x-ua-compatible 属性指定兼容模式为 IE8，目的是使页面按照 IE8 的模式进行渲染，进而模拟在低版本 IE 浏览器中的测试。有关这一属性的理解可参考文章 IE 兼容模式（x-ua-compatible）
