汇总一些 HTTP 常见的面试题

1. HTTP 协议的主要特点

  • 简单快速 （每个资源的 URL 是固定的，只需通过 URL 即可访问）
  • 灵活（可以传输不同类型的数据，通过报文头部的 content-type 字段设置数据格式）
  • 无连接（一次连接后会自动断开，不会保持连接）
  • 无状态（每次 HTTP 请求都是独立的，默认不会保存上下文信息。通过 cookie 可以实现有状态的连接）

2.HTTP 报文头部组成部分

  1. 请求报文头部
  • 请求行
    包含请求方法，请求地址，HTTP 协议版本
  • 请求头
    包含各种头部字段（如 Host、Connection、Cookie 等），也可以携带自定义头部字段
  • 空行
    告诉服务端请求头结束，下面是请求体部分
  • 请求体
    放置请求数据
  1. 响应报文头部
  • 状态行
    包含 HTTP 协议版本，响应状态码
  • 响应头
    包含各种头部字段（如 Server、Content-Type、Content-Length 等）
  • 空行
    告诉客户端响应头结束，下面是响应体部分
  • 请求体
    放置响应的信息

3. HTTP 常用的请求方式

  • GET 向指定的 URL 请求数据
  • POST 向服务器提交数据（如提交表单或上传文件）
  • HEAD 跟 GET 类似，但响应中只返回报文头部
  • PUT 提交数据替代指定内容
  • DELETE 请求服务器删除指定内容
  • OPTIONS 获取允许的请求方式
    …

更多请参考HTTP 请求方法

4. HTTP 常用的状态码

  • 1XX：表示接受的请求正在处理
  • 2XX：表示请求正常处理完成
  • 3XX：表示资源重定向，需要重新请求
  • 4XX：表示客户端请求错误，服务器无法处理
  • 5XX：表示服务端处理请求出现错误

常用状态码
101 请求协议切换，请 HTTP 切换到 WebSockt
200 请求成功，正常返回消息
301 资源永久重定向，即请求的资源永久迁移到了其他地址
302 资源临时重定向，请求的资源临时转移至新的 URL
304 请求的资源未更新，继续使用本地缓存
400 客户端请求有语法错误，服务端无法理解
403 服务端禁止访问
404 未找到请求的资源
405 请求方法不被允许
500 服务器内部错误，无法完成请求
502 网关错误

更多请参考HTTP 状态码

5. GET 和 POST 请求的区别

  • GET 请求默认会被浏览器缓存，而 POST 请求需要手动开启
  • GET 请求回退时是无害的，不会重新发起请求，而 POST 会再次请求数据
  • GET 请求的参数会保存在浏览器历史记录中，而 POST 不会
  • GET 请求的参数直接放在 URL 中，通过&连接，POST 请求的参数则放在请求体内
  • GET 请求在 URL 中传递的参数长度有限制（各浏览器中标准不同），而 POST 没有限制
  • GET 请求只支持 URL 编码，而 POST 支持多种编码方式
  • GET 请求的参数只能是 ASCII 格式的数据，POST 请求的参数可以使用任何格式的数据
  • GET 请求比 POST 请求不安全，请求参数直接暴露在 URL 中，不能用来传输敏感数据
  • GET 请求产生的 URL 可以被浏览器收藏，而 post 不可以

6. HTTP 持久连接（长连接）

HTTP 连接可以分为短连接和长连接，由于 HTTP 协议采用“请求-应答”模式，每次请求/响应都要在客户端和服务端之间新建一个连接，完成之后立即断开连接（因为 HTTP 是无连接的协议），这属于短连接。长连接就是指使用 Keep-Alive ，使客户端到服务端的连接持续有效，避免了当后续出现请求时需要重新建立连接。

HTTP 长连接的实质是 TCP 长连接，所谓的长连接实际上是在 TCP 层实现的，TCP 连接在通讯双方之间建立了一个全双工通信通道，通过复用一个 TCP 连接可以持续进行通信，也就实现了 HTTP 的长连接。

实现方式
要实现 HTTP 持久连接，只需服务端在响应头中设置Connection：Keep-alive即可。HTTP1.1 默认是长连接，HTTP1.0 也支持长连接，但需要手动开启。

7. HTTP 管线化

简单来说就是客户端将多个 HTTP 请求打包为一个发送给服务端，服务端也可以将多个响应打包成一个返回给客户端（服务端不一定要进行管线化）。

特点

  • 管线化依赖于持久连接，仅 HTTP/1.1 支持
  • 只有 get 和 head 请求可以进行管线化，post 有所限制
  • 初次创建连接时不应启动管线化，因为请求的服务器不一定支持 HTTP / 1.1
  • HTTP / 1.1 要求服务器端支持管线化，但并不要求服务器端也对响应进行管线化处理
  • 管线化不会影响响应返回的顺序（服务端未对响应进行管线化时）
  • 开启管线化可能并不会带来大幅度的性能提升，由于很多服务器端和代理程序对管线化的支持并不好，因此现代浏览器如 Chrome 和 Firefox 默认并未开启管线化支持

8. HTTP 和 HTTPS 的区别

HTTPS（Hyper Text Transfer Protocol over Secure Socket Layer）是以安全为目标的 HTTP 通道，在 HTTP 的基础上通过传输加密和身份认证保证了传输过程的安全性，简单来说就是 HTTP 的安全版。

HTTPS 的作用

  • 保证数据传输的安全性
  • 确认网站的真实性，防止钓鱼网站

HTTP 与 HTTPS 的区别

  • HTTP 是明文传输，HTTPS 通过 SSL\TLS层 对报文进行了加密；
  • HTTP 的端口号是 80，HTTPS 是 443；
  • HTTPS 协议相比 HTTP 多了一个 SSL 层，可进行数据加密、完整性校验、身份验证，比 HTTP 协议要安全；
  • HTTPS 需要到 CA（Certificate Authority） 申请证书，一般免费证书很少，需要购买；
  • HTTPS 由于需要进行加密、解密操作，因此性能消耗会大于 HTTP；

HTTPS 的原理
HTTPS 的安全性通过介于 HTTP 层和 TCP 层之间的 SSL 层实现，具体如下：

  • 数据加密
    HTTPS 采用非对称加密+对称加密结合的方式，对传输数据进行加密。
  • 身份验证
    通过CA颁布的数字证书进行身份验证，保证当前服务器的真实性。
