1. 如何实现路由懒加载

在打包构建应用时，随着组件的增多，打包出来的js文件会越来越大，有时候我们需要在指定路由下才加载对应的组件，从而优化应用的首屏加载速度，这就是所谓的路由懒加载，结合 Vue 的异步组件和 Webpack 的代码分割功能就能实现。

原理

  1. 首先将异步组件定义为返回一个 Promise 的工厂函数 (该函数返回的 Promise 应该 resolve 组件本身)
  2. 在 Webpack 中，使用动态 import 语法来定义代码分块点 (split point)

结合两者，就能定义一个能够被webpack自动分割代码的异步组件

实现

如果需要将某个路由下的所有组件都打包在同一个异步块（chunk）中，可以使用命名chunk，用特殊的注释语法来提供 chunk name

2. 路由安全

vue-router中保护路由安全通常使用路由守卫，通过设置路由导航钩子函数的方式添加路由守卫,并在钩子函数里判断用户的登录状态和权限，从而达到保护指定路由的目的。

具体实现有几个层级：
全局前置守卫beforeEach、路由独享守卫beforeEnter和组件内守卫beforeRouteEnter。

以全局守卫为例来说，可以使用router.beforeEach((to,from,next)=>{})方式设置守卫。每次路由导航时，都会执行该守卫，从而检查当前用户是否可以继续导航。

通过给其中的next函数传递多种参数达到不同的目的，比如要禁止用户继续导航可以传递next(false)，正常放行可以不传递参数，传递path字符串可以重定向到一个新的地址等等。

这些钩子函数之所以能够生效，和vue-router工作方式有关。像beforeEach只是注册一个hook，当路由发生变化，router准备导航之前会批量执行这些hooks，并且把目标路由to、当前路由from以及后续处理函数next传递给我们设置的hook。
