模块化
Eli Lv4
  2021-05-19 21:19:14 2021-05-19 21:19  
  • JS 
 
  • JS 

JS 起初没有自带的模块体系，无法将程序拆分成相互依赖的小文件再组织到一起，在 ES6 之前，JS 社区提出了 commonJS、AMD 等规范作为模块化解决方案，前者作用于服务端，后者作用于浏览器，而 ES6 则在 JS 语言的层面上实现了模块体系。

1. commonJS 规范

commonJS 规范主要是为了解决 js 的作用域问题而定义的模块形式，可以使每个模块在其自身的命名空间中执行。通过 module.exports 暴露出对外的变量和接口，通过 require 命令将其他模块的输出导入到当前模块的作用域中。

Nodejs 早期使用的就是 commonJS 规范， 从 8.5.0 开始原生支持 ES 模块，低版本中可以使用 babel 来进行兼容。

2. AMD 规范

AMD(Asynchronous Module Definition)翻译为异步模块定义，采用异步方式进行模块加载，即在加载模块时不影响后面语句的执行。AMD 也采用 require()语句加载模块，但是不同于 CommonJS，它接收两个参数；

3. ES Module

  • export 命令

export 命令规定了模块内部变量、方法对外的接口，将模块内部的变量和方法暴露出去。

  • import 和 require 命令
– import require
加载方式 静态加载 动态加载
加载时间 编译时 运行时

这两个命令都可以用于导入外部模块，import 所谓的静态加载，指的是在 JS 引擎编译过程中就进行了外部模块的加载；相应的，require 动态加载指的是在程序执行过程中才进行外部模块加载。这样带来的问题是—— require 可以实现按需加载，而 import 则不能。

ES6 给出了解决方案，使用 import() 就可实现动态加载，import() 可以接收 import 命令所能接受的一切参数，返回一个 Promise 对象。import() 和 require 的区别是，前者为异步加载，后者为同步加载。
