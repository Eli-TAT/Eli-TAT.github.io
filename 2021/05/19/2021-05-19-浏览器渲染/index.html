1. 网页渲染流程

主流浏览器内核（渲染引擎）

浏览器 内核 前缀
Chrome WebKit -webkit-
Safari WebKit -webkit-
Firefox Gecko -moz-
IE Trident -ms-
Opera Presto -o-

浏览器渲染引擎渲染网页的过程主要分为以下几步，webkit 引擎和 Gecko 引擎的渲染流程基本相同。

  1. 构建 DOM 树（parse）：渲染引擎解析 HTML 文档，将 HTML 标签转换成 DOM 树中的 DOM 节点（包括 js 生成的标签），生成 DOM 树；
  2. 构建 CSS 规则树：解析对应的 CSS 样式文件（包括 js 生成的样式和外部 css 文件），生成 CSS 规则树；
  3. 构建渲染树（construct）：合并 DOM 和 CSSOM，构建渲染树（Rendering Tree/Frame Tree）；
  4. 布局渲染树（layout）：从渲染树的根节点开始遍历，计算每一个元素的大小、位置等，给出每个节点应在屏幕上出现的坐标；
  5. 绘制渲染树（paint）：遍历渲染树，将各个节点绘制到屏幕上；

2. 重绘与重排

重排(回流)一定引起重绘，重绘不一定引起重排(回流)

  1. 重排(reflow)
    重排指的是当渲染树的一部分必须更新并且节点的尺寸发生了变化时，浏览器会使渲染树中受到影响的部分失效，并重新构造渲染树。

引发重排的操作

  • 页面初次渲染(无法避免)
  • 添加或删除可见的 DOM 元素
  • 元素位置的改变，或者使用动画
  • 元素尺寸的改变，如宽高、外边距、边框
  • 浏览器窗口尺寸（resize 事件）和默认字体的变化
  • 填充内容的改变，比如文本的改变或图片大小改变而引起了元素宽度和高度的改变
  • 读取某些元素属性：(offsetLeft/Top/Height/Width,clientTop/Left/Width/Height,scrollTop/Left/Width/Height,width/height,getComputedStyle(),IE 的 currentStyle()等等)
  1. 重绘(repaint)
    重绘指的是当元素外观发生改变时，浏览器会根据元素新属性重新绘制，使元素呈现新的外观

引发重绘的操作

  • 如改变元素字体颜色、背景颜色等等

由于重绘和重排耗费性能和时间，会导致浏览器卡顿，影响加载效率，因此开发过程中应尽量减少重绘和重排。

优化策略

（1）浏览器自身的优化策略
浏览器会维护一个队列，把所有会引起回流、重绘的操作放入其中，等队列中的操作到了一定的数量或者到了一定的时间间隔，浏览器就会进行一次批处理，这样就让多次的回流、重绘变为了一次。
（2） 开发者的优化策略
减少重绘和重排实际上就是减少对渲染树的操作，可以合并多次的 DOM 修改 和样式修改，并减少对 style 属性的访问。

  • 预先定义好 css，直接修改元素的 className 而不是操作 style 属性；
  • 不要经常访问会让浏览器 flush 队列的属性，如果一定要访问，可以将需要访问的值存储起来，再次使用就不会再引发回流；
  • 对于需要多次重排的元素，position 属性设为 absolute 或 fixed，元素脱离了文档流，其变化不会影响到其他元素；
  • 需要创建多个 DOM 节点时，可以使用 DocumentFragment 创建完后一次性插入 document；
  • 尽量不要使用 table 布局；
  • 使用 cloneNode()和 replaceChild()，只会引发一次回流和重绘；
