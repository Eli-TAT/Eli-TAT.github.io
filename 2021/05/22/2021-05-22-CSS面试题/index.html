汇总一些 CSS 面试题

1. 盒模型

所有 HTML 元素都可以看做一个盒子，CSS 盒模型将元素定义为一种由内容、边框、内边距、外边距构成的盒子，使用盒模型可以对网页中的元素进行快速排版和布局。

概念

  • content
    元素的内容区域
  • border
    元素的边框
  • padding
    负责控制元素内部空出的空间，也就是内边距，会改变元素实际大小，不能取负值。背景色会延伸到 padding 部分，且 padding 对背景图片无影响。
  • margin
    负责控制元素外部空出的空间，也就是外边距，不影响元素实际宽高，只改变盒模型所占空间大小。margin: 0 auto可用于设置元素水平居中。
    上下两个元素的外边距有时会发生重叠，实际空出的空间取两外边距中的较大值（外边距重叠问题可通过 BFC 解决）。

分类
CSS 盒模型分为标准盒模型以及怪异盒模型（IE 盒模型）两种，两者的区别在于元素大小的计算方式。通过 box-sizing 属性可以切换盒模型。

  • 标准盒模型box-sizing:border-box
    元素大小=content+padding+border，也就是说 width=content+padding+border
  • 怪异盒模型box-sizing:content-box
    元素大小=content，也就是说 width=content

标准盒模型设置 width 后元素 content 的实际大小还要减去 padding 和 border，不仅容易造成混乱而且后续修改会带来一些麻烦，相比之下怪异盒模型显得更为方便，因此开发中使用怪异盒模型较多。

2. position 属性

position 定位属性指定了元素在文档流中的定位方式，有五个取值。

  1. static

position 属性的默认值，表示元素遵循正常文档流，没有定位。除 static 外，position 属性为其他四个值的元素被称为定位元素

  1. relative(相对定位)
  • 相对定位元素不脱离文档流
  • 可以使用偏移属性(top、left、right、bottom)进行定位，但仍然占据元素偏移前所占据的空间
  • 经常被用作为绝对定位元素的容器块
  1. absolute(绝对定位)
  • absolute 绝对定位元素脱离文档流
  • 可以使用偏移属性进行定位，不占据元素偏移前所占据的空间
  • 以最近的祖先定位元素（非 static，一般是 relative）的左上角作为原点进行偏移，若没有相对定位元素作为基准，则以浏览器窗口的左上角作为原点进行偏移
  1. fixed(绝对定位)
  • fixed 绝对定位元素脱离文档流
  • 可以使用偏移属性进行定位，不占据元素偏移前所占据的空间
  • 以浏览器窗口的左上角作为原点进行偏移
  1. sticky(粘性定位)
  • 基于用户的滚动位置来定位，跨越特定阈值(偏移属性)前元素表现为相对定位（relative），跨越后表现为固定定位(fixed)
  • 若不设置阈值则不会触发粘性定位，等同于相对定位
  • 不兼容 IE 浏览器，safari 中使用需加前缀

元素的定位与文档流无关，所以元素可以覆盖页面上的其它元素，z-index 堆叠属性指定了一个元素的堆叠顺序（哪个元素应该放在前面，或后面）

3. 隐藏元素的三种方式

它们的区别主要体现在：元素是否占据页面空间、能否添加事件监听、是否引起重绘重排、是否有过渡效果、是否能被继承。

  1. display:none
  • 设置了该属性的元素不会被浏览器渲染，并且不会占据页面空间（在 DOM 树中，但不在渲染树中）
  • 不能对设置了该属性的元素添加事件监听
  • 会引起重排，因为页面布局发生改变
  • 无法设置过渡效果，transition: display无效
  • 不会被子元素继承(子元素设置display: block不会显示)
  1. visibility:hidden
  • 不会被浏览器渲染，但会占据页面空间（既在 DOM 树中，也在渲染树中）
  • 不能添加事件监听
  • 只引起重绘，因为元素位置未发生改变
  • 可以设置过渡效果，transition:visibility
  • 可以被继承,子元素设置visibility:visible可以显示
  1. opacity:0
  • 会被浏览器渲染，且会占据页面空间，只是被隐藏（既在 DOM 树中，也在渲染树中）
  • 可以添加事件监听
  • 不会引起重绘或重排
  • 可以设置过渡效果,transition: opacity
  • 可以被继承, 子元素设置 opacity!=0 可以显示
  • opacity 可以触发硬件加速

CSS3 硬件加速实际上是在布局渲染树(layout)阶段将触发了硬件加速的元素提升到独立的图层，使用 GPU 进行渲染，这样就避免了动效引起的重绘，节约了浏览器性能。

深入了解 css3 硬件加速

兼容 opacity 属性
在浏览器中设置不透明度需要进行兼容处理，其中 IE 完全不支持 opacity 属性，只能通过 filter 滤镜属性来进行模拟。具体兼容方式可参考CSS 透明 opacity 和 IE 各版本透明度滤镜 filter 的最准确用法

其他隐藏元素的方式

  • 设置 z-index 为负值；
  • 设置 margin 让元素离开可视区域；

5. BFC

BFC 全称块级格式上下文，规定了容器内的布局规则。详细内容请看 MDN

BFC 的特点

  • 设置了 BFC 的容器不会出现垂直方向外边距重叠的问题
  • BFC 是一个独立的容器，内外的元素不会相互影响
  • 设置了 BFC 的容器不会与浮动元素重叠
  • BFC 在计算高度时会把内部浮动元素的高度也计算在内

如何设置 BFC

  • 设置元素浮动，float 取 none 以外的值
  • 设置元素绝对定位，position 取 absolute 或 fixed
  • 设置行内块级元素，行内元素设置 display 为 inline-block
  • 设置表格元素，display 属性取任何包含 table 的属性值
  • 设置块元素 overflow 不为 visible，为 hidden/auto/scroll
  • 弹性元素（flex 容器的直接子元素）
  • 网格元素（grid 容器的直接子元素）
